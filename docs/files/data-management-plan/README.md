# NMCB Data Management Plan (DMP)

## 📁 Folder Overview

This folder contains the **NMCB Data Management Plan (DMP)** and related materials used to support data management across projects.

It serves as the central reference point for:

- The main NMCB data management framework
- Templates for sub-projects
- Supporting training materials

---

## 📄 Contents

### 1. Main DMP

- **File:** `NMCB Data Management Plan Version 2.0.pdf`
- **Description:**  
  The official NMCB Data Management Plan.  
  This document outlines the overall data governance, storage, access, and sharing principles.

---

### 2. Sub-project DMP Template

- **File:** `NMCB Data Management Plan Template for sub-projects.docx`
- **Description:**  
  Template to be completed for each new sub-project.
- **Important:**  
  Many sections in this template are designed to **refer to the main NMCB DMP**, avoiding duplication and ensuring consistency.
- Examples: `VIRGIMME`, `MALIBU`, `IDO2 Pathway in Post-COVID Project`

---

### 3. Training Materials

- **Folders:** `Training materials`
- **Description:**  
  Supporting materials and examples that help users understand how to complete and apply the DMP.

---

## 🚀 How to Use This Folder

When a **new project** starts:

1. Share the following documents with the project team:
   - Main DMP (`.pdf`)
   - Sub-project DMP template (`.docx`)
2. Instruct the team to:
   - Use the template to document project-specific details
   - Refer to the main DMP where applicable (do not duplicate content)
3. Optionally:
   - Share relevant training materials from this folder

---

## 💡 Notes

- The main DMP is the **single source of truth** for NMCB data management.
- Sub-project DMPs should remain **aligned** with the main DMP.
- Keeping references instead of duplications ensures easier updates and consistency.
