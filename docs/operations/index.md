# Operations overview

The operational side of the role keeps the study running day to day. It includes recurring coordination tasks and a set of small but critical maintenance activities.

## Core recurring tasks from the board

- weekly NMCB mailbox review
- weekly visit log maintenance
- bi-weekly subject ID log maintenance
- bi-weekly sending of screeners to Linked2Trial
- sample requests (as received)
- get numbers (as requested; may be weekly or monthly for routine reports)
- preparation of a routine checklist for recurring data tasks

## Operational design principle

A recurring task should be documented in a way that makes it easy to execute and easy to audit. Each task page should include:

- purpose
- trigger or frequency
- inputs
- steps
- output
- quality checks
- escalation route
